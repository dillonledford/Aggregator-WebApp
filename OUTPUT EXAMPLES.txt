HKUDS/DeepTutor

This project has seen a rapid release cadence with multiple updates focusing on core AI capabilities, user experience, and platform compatibility.

Version DeepTutor-v1.1.0

    Key Enhancements: This release includes significant overhauls in LaTeX parsing, LLM configuration, and API compatibility.
        Feature: Overhauled LaTeX block math parsing to correctly handle loose block math and improve delimiter detection.
        Configuration Change: Moved LLM probe max_tokens setting to agents.yaml for unified configuration.
        API Improvement: LLM factory functions (llm_complete, llm_stream) now correctly forward extra_headers.
        Bug Fix: Resolved an issue with saving to notebooks due to UUID mismatches in API calls.
        Documentation Update: Improved guidance on using host.docker.internal for local LLM setups.
        Testing: Expanded the test suite to cover new features and bug fixes.
        Community Contributions: Notable contributions from @DarkGenius and @sagnikonly.

Version DeepTutor-v1.1.0-beta

    Key Features: This beta release introduces major changes to chat routing, theming, and real-time communication.
        Feature: Migrated chat experience to URL-based routing (/chat/[[...sessionId]]) making sessions bookmarkable.
        New Feature: Added a Snow theme to the available UI themes.
        Real-time Improvement: Implemented WebSocket heartbeat and auto-reconnect for more stable connections.
        Feature: Introduced a streaming idle timeout to automatically handle stale sessions.
        Performance: Optimized ChatComposer to fix severe input lag in long conversations.
        Configuration: Overhauled the Embedding Provider Registry for better type safety and UI integration.
        Feature: Restored Serper Search Provider.
        Resilience: Improved resilience in the Deep Research Reporting Agent by falling back to raw LLM output on JSON parsing errors.
        Bug Fix: Corrected Markdown renderer's inline vs. block code detection.
        Compatibility: Added support for Ollama's embedding response format.
        Development Workflow: Added auto-generation of .env.local for the launcher.
        Testing: Expanded test suite with modules for various core components.

Version DeepTutor-v1.0.3

    Key Features: This release focuses on knowledge management, visualization, and model compatibility.
        New Feature: Introduced Question Notebook for unified quiz review with metadata, bookmarking, and categorization.
        New Feature: Added Mermaid Diagram Support for visualizations (flowcharts, sequence diagrams, etc.).
        Feature: Implemented Embedding Model Mismatch Detection for knowledge bases, flagging KBs that need re-indexing.
        Compatibility Improvement: Merged system messages for better compatibility with Qwen models served via vLLM.
        Provider Support: Added first-class support for LM Studio and llama.cpp as LLM providers.
        New Theme: Introduced the Glass theme.
        Resilience: Enhanced Deep Research Reporting Agent resilience with retries.
        Documentation: Migrated documentation from the legacy VitePress folder.
        Community Contributions: Notable contributions from @cskwork, @OlegSob-glitch, and @SuperMarioYL.

Version DeepTutor-v1.0.2

    Key Enhancements: This release addresses search provider handling, resource leak fixes, and Docker build improvements.
        Search Consolidation: Simplified search consolidation and added SearXNG fallback.
        Bug Fix: Ensured base_url is correctly overwritten when switching providers.
        Testing Improvement: Explicit runtime configuration in the Test Runner for accurate test execution.
        Resource Leak Fixes: Addressed leaks in Playground testers, UnifiedChatContext session memory, WebSocket runners, and auto-scroll timers.
        Bug Fix: Resolved a Docker build issue related to setTimeout return types.
        Community Contributions: Notable contribution from @OlegSob-glitch.

Version DeepTutor-v1.0.1

    Key Features: Introduces a new Visualize capability and improves chat composer interaction.
        New Feature: Added Visualize capability for generating Chart.js or SVG visualizations from data descriptions.
        Feature: Introduced an explicit Reference Picker in the chat composer for attaching notebook records or chat history.
        Bug Fix: Prevented duplicate quiz questions by implementing a previous_questions parameter.
        Compatibility: Added support for o4-mini and future o-series models.
        Logging Improvements: Suppressed noisy uvicorn logs and added selective HTTP access logging.
        Community Contributions: Several contributors listed for various fixes and features.

Version DeepTutor-1.0.0-beta.4

    Key Enhancements: Focuses on embedding progress tracking, cross-platform compatibility, and MIME validation.
        Feature: Added real-time embedding progress reporting during knowledge base initialization.
        Feature: Implemented HTTP 429 rate limit retries with exponential back-off for embedding.
        Compatibility: Improved cross-platform start tour dependency management (macOS, Linux, Windows).
        Bug Fix: Fixed case-insensitive MIME type validation for file extensions on Linux.
        Community Contributions: Contributions from @oxkage and @kuishou68.

Version v1.0.0-beta.3

    Key Changes: This release removes the litellm dependency, improves Windows compatibility, and enhances JSON parsing robustness.
        Dependency Removal: Replaced litellm with native openai and anthropic SDKs, introducing OpenAICompatProvider and AnthropicProvider.
        Bug Fix: Fixed Math Animator compatibility on Windows using subprocess.Popen and asyncio.Queue.
        Robustness: Implemented parse_json_response() in seven agent modules for safer LLM JSON output handling.
        Guided Learning Fixes: Corrected KaTeX math rendering and backend polling logic.
        Internationalization: Completed i18n coverage for the web UI (English and Chinese).
        Community Contributions: Contributions from @kagura-agent, @kevinmw, and @LocNguyenSGU.
